# LocalWrap v3.1 Trust, Polish, Habit

## Status

- Stage: Implementation note
- Target release: v3.1.0
- Last updated: 2026-06-16

## Product Intent

v3.1 turns the v3 localhost cockpit into a more trustworthy daily tool. The
release should make LocalWrap look more polished, make workspace resume more
intentional, and tighten the release gates around the golden path.

## Included

- Modern retro control-room renderer polish: compact desktop utility, clearer
  state surfaces, fewer literal Win95 bevels, stronger preview hierarchy.
- Named workspace profiles persisted in `projects.json` under workspace
  metadata.
- Required Electron smoke tests in CI, plus release-time lint, format, and smoke
  verification before publishing installers.
- Async quit cleanup that waits for running project shutdown.

## Release Trust Track

Code signing remains a follow-on business/process task rather than a hidden
technical dependency in v3.1. Track these before a broader public push:

- Apple Developer ID certificate and notarization credentials for signed macOS
  DMGs and reliable macOS auto-update.
- Windows Authenticode certificate and SmartScreen reputation plan for NSIS
  installers.
- Manual install-and-launch checks for macOS DMG, Windows EXE, and Linux
  AppImage artifacts before publishing a GitHub Release.

## Deferred

- Portable repo configs.
- Plugin/detector architecture.
- Cloud sync or team workspaces.
- One-click dependency installation or framework repair automation.
