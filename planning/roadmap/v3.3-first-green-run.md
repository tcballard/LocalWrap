# LocalWrap v3.3 First Green Run

## Status

- Stage: Implementation note
- Target release: v3.3.0
- Last updated: 2026-06-23

## Product Intent

v3.3 helps a developer understand why a repo-backed workspace is or is not ready
to start. v3.2 let a repository describe the local stack; v3.3 adds the stack
preflight that makes the first green run feel explained instead of guessed.

## Included

- Workspace Doctor diagnosis for the selected workspace profile, last running
  set, or all saved projects.
- Cross-project checks for missing directories, unsafe commands, missing
  `node_modules`, `.env.example` gaps, duplicate ports, busy ports, and local URL
  validity.
- Per-project ready, attention, and blocked summaries above Project Setup.
- **Start Ready** action that starts only projects without Workspace Doctor
  blockers and skips blocked services.
- IPC and preload methods for workspace diagnosis and start-ready behavior.
- Unit and renderer anchor coverage for the Workspace Doctor surface.

## Product Bar

- A developer can import a repo workspace pack and immediately see which services
  are ready, which need attention, and which are blocked.
- LocalWrap avoids starting projects with known blocking configuration failures.
- Warnings stay visible without preventing partial progress.

## Deferred

- Explicit service dependency graph and startup ordering.
- Structured health-check definitions in workspace packs.
- One-click install or env file generation.
- Runtime/toolchain checks beyond local files and ports.
- Visual redesign or enterprise aesthetic refresh.
