# LocalWrap v3.2 Team-Ready Workspaces

## Status

- Stage: Implementation note
- Target release: v3.2.0
- Last updated: 2026-06-19

## Product Intent

v3.2 starts turning LocalWrap from a personal localhost cockpit into a
repo-carried team tool. A repository should be able to explain the local stack it
needs, let a developer import that stack safely, and make the first green run a
reviewable one-click path rather than tribal setup knowledge.

## Included

- Portable workspace pack discovery at `.localwrap/workspace.json`, with
  fallback support for `localwrap.json`.
- Workspace pack import that saves projects and named workspace profiles without
  starting commands automatically.
- Workspace pack export from saved LocalWrap projects into a repo-relative JSON
  file.
- Pack validation through the existing command allowlist, local URL checks, and
  safe relative project paths that cannot escape the selected workspace folder.
- Source metadata on imported projects and profiles so re-importing the same
  pack updates existing LocalWrap records instead of duplicating them.

## Product Bar

- A developer can clone a repo, choose **Import Workspace**, review the detected
  project commands, and save the stack without touching a terminal.
- A maintainer can configure the stack once in LocalWrap, choose
  **Export Workspace**, and commit the generated `.localwrap/workspace.json`.
- Import remains safe by default: LocalWrap never starts imported commands until
  the developer explicitly starts or resumes the workspace.

## Deferred

- JSON Schema publishing for editor validation.
- Framework/runtime requirement checks beyond the current Project Doctor
  coverage.
- Dependency installation or repair automation.
- Team/cloud sync.
- Plugin-based detector architecture.
